2. Data Type + Type System

a. Why NaN is number ?
typeof NaN === 'number' // true

Ans:
NaN isn't a separate type — it's a special value within JavaScript's number type, defined by the IEEE 754 floating-point standard as the result when a math operation fails to produce a valid number (like 0/0 or Math.sqrt(-1)). It's still stored as a number internally, just an "invalid" one — so typeof correctly reports number.
Bonus: NaN !== NaN is also part of that same IEEE 754 spec, which is why you check for it with Number.isNaN(x) instead of ===.

b. undefine vs null
let x;
console.log(x);     // undefine
let y = null;
console.log(y);     // null

Ans:
undefined — a variable has been declared but not yet assigned a value. JavaScript sets this automatically; you didn't ask for anything.
null — represents "no value" or "empty," but it's intentional. A developer explicitly assigns it to say "this is deliberately empty."
let x;          // declared, never assigned → undefined (JS's doing)
let y = null;   // explicitly set to "nothing" → null (your doing)

c. "5" + 1, "5" - 1
'5' + 1 // "51"
'5' - 1 // 4

Ans:
+ does string concatenation when either side is a string. JavaScript sees "5" + 1, notices one operand is a string, and decides "okay, this is concatenation" — converts 1 to "1" and glues them together: "51".
- only ever means subtraction. There's no such thing as "string subtraction," so JS has no ambiguity to resolve — it just forces both sides into numbers and subtracts: "5" - 1 → 5 - 1 → 4.
So the rule of thumb: + is overloaded (math or concatenation, decided by operand types), while -, *, / always coerce to numbers since they only have one meaning.
"5" + 1   // "51" (concatenation wins)
"5" - 1   // 4    (forced to number)
"5" * 2   // 10   (forced to number)
1 + "1" + 1   // "111" (left to right: 1+"1"="11", then "11"+1="111")
1 + 1 + "1"   // "21"  (1+1=2 first, then "2"+"1"="21")
